## 创建开发环境

首先进入 Sealos 桌面：**https://hzh.sealos.run**

然后打开 Debox 应用，创建一个新项目，这里我们选择使用 Next.js 作为运行环境，因为 Next.js 是现在最流行的前端框架之一嘛，我们就用它来开发一个网站。

![](https://images.tryfastgpt.ai/forum/2024-11-20-13-52-COZ4XF.png)

由于 Sealos Devbox 目前还没有完全适配 Windsurf（我们正在快马加鞭支持中），所以这里我们可以先曲线救国，使用 VS Code 或者 Cursor 来连接开发环境，连接成功后，再关闭 VS Code，然后打开 Windsurf 就可以连接到开发环境了。

具体步骤如下：

## 开发网站

我们先来看一下右边的这个 Cascade 面板：

![](https://images.tryfastgpt.ai/forum/2024-11-20-14-04-tf3xk0.png)

这个面板是 Windsurf 的特色功能，和 Cursor 的 Composer 类似，但 Cascade 的操作更灵活一点。

Cursor 需要在 Chat 面板和 Composer 面板之间来回切换，而 Windsurf 则只需要在 Cascade 面板中切换模式即可。Cascade 提供了**聊天**和**编辑**两种模式，聊天模式和 Cursor 的 Chat 面板类似，而编辑模式则和 Cursor 的 Composer 面板类似，可以生成和修改代码。

Cascade 这里的设计有点误导，我一开始以为把切换按钮打开就是编辑模式，实际上不是这个意思，切到左边是 “Write”（也就是编辑模式），切到右边是 Chat（也就是聊天模式）。

![](https://images.tryfastgpt.ai/forum/2024-11-20-14-09-lyAgsN.png)

至于模型嘛，大家都一样，默认当然是最强的 Claude 3.5。

我们可以使用 Sealos AI Proxy 中的模型，这是一个一站式 AI 模型调用解决方案，让你可以在同一个平台中轻松调用各类 AI 模型。

在 Sealos 桌面打开【AI Proxy】，点击【新建】按钮开始新建 Key。

创建完成后会看到 API Endpoint 和需要调用的 API Key，直接点击复制进行使用。

![](https://images.tryfastgpt.ai/forum/2024-11-18-00-10-Mji6aD.png)
